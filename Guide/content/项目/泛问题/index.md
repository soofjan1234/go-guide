## 印象最深刻、最出彩的项目/解决了什么问题/遇到什么坑

线上有用户反馈：往 NAS 里丢一批大文档（30MB 以上的 TXT）后，文档中心无响应。我会先确认影响范围和现象：是整个 NAS 死机、文档中心服务退出、页面卡死。

然后我们穿透过去看日志系统，发现我的这个应用内存占用率飙高。

我先在自己的测试机复现，用 pprof 对 Runtime 做 Heap 采样，确认瓶颈在 Bleve 建索引峰值很高。

查官方配置和AI调参效果有限，于是深入读了 Bleve 索引构建源码。

最后落地一套方案：收紧 Bleve 配置减小索引体积；大文档按约 1MB 分块写入；每累计约 10MB 再 Batch 提交一次。

压测对比 20 多种分块/Batch 组合后选定，峰值 heapInuse 降幅约 76%。

## Docker

Docker 项目主要封装 Docker Engine API，提供镜像、容器、网络和基础状态管理能力。比如查询本地镜像、删除镜像、创建容器、启停容器、查询容器状态、端口映射、挂载目录、环境变量配置等。

镜像拉取这里做了任务化处理。前端先调用接口生成 pullID，后端为这个 pullID 创建 context 和 cancel function。真正拉取时调用 Docker API 的 ImagePull，Docker daemon 会返回 JSON stream，我们解析这个 stream，把拉取状态通过 SSE 实时推给前端。如果用户取消，就通过 pullID 找到对应的 cancel function，取消 context 并清理任务。
