## 印象最深刻、最出彩的项目/解决了什么问题/遇到什么坑

我印象最深的是文档中心的内存优化问题。

项目刚上线时，有用户反馈导入 50MB 的 TXT 文档后，文档中心直接无响应。我先确认了问题现象，然后在测试机复现，用 top 发现内存飙升导致应用崩溃。接着用 pprof 定位，发现瓶颈在 Bleve 建立索引阶段。

查官方文档和 AI 建议效果都不太好，我就去读了 Bleve 的源码。最后通过三个方案解决：一是调整配置，比如禁用 _all 字段来减小索引体积；二是文档分块，1MB 一块；三是批量提交，累计 10MB 再写入。

最终效果很明显，峰值 heapInuse 降了 76%，彻底解决了这个问题

## Docker

Docker 项目主要封装 Docker Engine API，提供镜像、容器、网络和基础状态管理能力。比如查询本地镜像、删除镜像、创建容器、启停容器、查询容器状态、端口映射、挂载目录、环境变量配置等。

拉镜像是长耗时任务，所以接口先创建任务并返回 taskId，后端异步执行 docker pull，同时读取 Docker daemon 返回的 JSON stream。前端通过 SSE 订阅这个 taskId，后端把每一段进度事件转成 SSE event 推给前端。

SSE 比轮询实时性好、实现简单；如果要双向控制才考虑 WebSocket
