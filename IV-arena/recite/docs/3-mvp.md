# Loom Memory MVP 拆分

## mvp1：可运行基础骨架 √
做什么：
- 搭建 `FastAPI` 后端和 `React + Vite` 前端
- 打通前后端通信（健康检查 + 基础接口）
- 完成基础页面布局（左侧导航 + 主内容区）

结果：
- 项目可本地启动，页面可访问

## mvp2：今日复习闭环 √
做什么：
- 提供今日待复习列表（按到期时间筛选）
- 支持作答 + 用户自评（A/B/C）
- 返回 AI 评分、推荐间隔、下次复习时间
- 提供题目操作：`延后 +15 天`、`删除`

结果：
- 完成“复习 -> 评分 -> 调度 -> 更新状态”闭环

问题和改进（mvp2.1）：
- 评估时机改为“最后一题统一提交评估”。
- 导航逻辑调整：
  - 仅最后一题显示“提交评估”。
  - 最后一题不显示“下一题”。
  - 除第一题外，每题显示“上一题”。
- `延后 +15 天`、`删除` 改为“待确认状态”：
  - 点击后仅标记状态，不立即执行。
  - 统一在“提交评估”时落库并生效。
  - 评估结果里展示每题最终状态（例如：`A`、`延后15天`、`已删除`）。

问题和改进（mvp2.2）：
- 评估前不显示结果，提交后以 `dialog` 弹窗展示本轮结果。
- 无复习题目时显示空状态页面文案，不展示答题区。
- `延后 +15 天`、`删除` 按钮移动到卡片右上角，与题目同层展示。
- 题目、答案、状态本地化保存（后端 `storage.json` 持久化）。
- 评估结果本地化保存（评估日志写入 `storage.json`）。

问题和改进（mvp2.3）：
- 评估结果已保存（`data/网络/review_logs.json`）。
- 题目存储改为 `data/网络/cards.json`。
- 候选题存储为 `data/网络/candidates.json`。
- 评估结果卡片已删除，评估前不显示结果，提交后仅用 `dialog` 展示本轮结果。
- `dialog` 已补充文字说明（题目数量与状态含义）。

问题和改进（mvp2.4）：
- 持久化文件收敛为 `data/网络/cards.json`。
- 评估结果展示增加1个答案，ai参考原有答案对我的答案的解析
- 也不需要candidates/review_logs部分

## mvp3：题库可见可查 √
目标：
- 让“题库”从占位页变成可用页面，用户能看到和检索所有题目。

做什么：
- 来源source要改为分类 已知的分类：Go/MySQL/Redis/网络/项目/其它 导入的时候ai自动解析分类
- 新增题库列表（问题、参考答案、分类、状态、下次复习时间）。
- 增加关键词搜索（问题/答案），包含匹配。
- 新增功能：可以删除和修改题目、答案
- 真物理删除：那就把后端 delete 行为改成移除卡片

验收标准：
- 导入并入库的新题可在题库页立即看到。
- 搜索关键词后能正确过滤结果。

不做什么：
- 没有deleted状态 因为删除后 会直接从cards.json中删除 不会保留
- 不做批量编辑。
- 不做标签系统和高级筛选。

问题和改进（mvp3.1）：
1. 导航高亮与路由同步：点击题库后导航高亮不再消失，并始终对应当前所在页面（支持返回/刷新后仍正确）。
2. 题库分类筛选补全：筛选项改为 `全部、Go、MySQL、Redis、网络、项目、其它`；选择某一分类后立刻按分类过滤题目（默认 `全部`）。
3. 题目卡片展示参考答案：每道题除显示问题外，还展示参考答案（默认折叠）；提供“展开答案/收起答案”切换，折叠态答案样式更浅更小，展开态显示完整答案。

## mvp4：评估结果沉淀 √
目标：
- 评估结果用于后续“统计评估/弱项改进建议”，并可在重启后继续被读取。

做什么：
- 数据目录调整：`data/question/` 放题库卡片（例如 `cards-网络.json`、`cards-Go.json`），`data/review/` 放历史评估记录。
- 增加统计评估功能（简化版）：对选定题目/题集输出 `AI 解析`，重点说明你的弱项在哪里，并给出改进建议。
- 评估记录写入 `data/review/`，供统计评估读取使用（不要求在前端提供历史回看）。

验收标准：
- 重启服务后统计评估仍可基于 `data/review/` 读取已落盘的评估数据。
- 统计评估输出字段完整：至少包含题目、你的答案、AI解析（含弱项定位）、改进建议。

不做什么：
- 不做历史评估回看功能（不提供历史记录列表/倒序浏览/回放）。
- 不做统计图表。
- 不做导出功能。

问题和改进（mvp4.1）：
- （已处理）提示词：强调 Markdown `##` / `###` 标题通常对应考点，需改写成独立面试问句，正文作答案素材；fallback 分块按标题切分。
- （已处理）候选 ID：`cand-{len+1}` 在已有编号不连续时可能重复键导致列表与字典不一致；改为按已有 `cand-*` 最大数字后缀 +1，入库 `card-*` 同理；列表按 ID 排序；导入页展示 `候选 ID` 便于对照请求。

问题和改进（mvp4.2）：
- 不使用ai来处理问题解析
  - 用程序删除###、##、**、*、`、```、行首 -，保留数字符号
  - 导入分成上中下板块，上是导入，中是处理结果，下是候选题审查：能够修改、删除、同意入库以及增加

问题和改进（mvp4.3）：
- 板块宽度充满剩余的空间
- 增加旁边加个完成解析：清空上面的东西

## mvp5
今日复习修改（已实现）
1. **公司模式**：无输入框，心里回忆后点记住/模糊/忘了，无 AI 打分，按自评与矩阵算下次复习（`POST /api/review/batch` 传 `mode: "company"`）。
2. **家庭模式**：作答 + 批量 AI 评估（`mode: "home"`）。
3. **默认按星期（本机日期）**：周一至周五自动 **公司模式**，周六、周日自动 **家庭模式**（前端 `getAutoReviewMode()`，无手动开关、不写 localStorage）。
